# OPC Codex Skills

本目录保存从 `easychen/opc-methodology` 提取并本地化存放的 OPC Codex Skills。

## Source

- Raw clone: `data/raw/opc-methodology/`
- Preserved source copy: `sources/skill-sources/opc-methodology/skills/`
- Machine index: `data/processed/opc-skills/skill-index.json`
- Content summary: `data/processed/opc-skills/skill-content-summary.md`

## Skills

| Skill | Purpose |
| --- | --- |
| `opc-orchestrator` | 编排完整一人企业方法论流程，判断阶段、恢复进度、调用后续 Skill。 |
| `opc-resource-audit` | 盘点一人企业创始人的 8 类资源、约束和硬边界。 |
| `opc-niche-positioning` | 用市场映射、客户细分和“三环合一”选择利基市场。 |
| `opc-value-proposition` | 为候选客群设计并选择价值主张。 |
| `opc-business-model-design` | 用 Lean Canvas 和简化商业模式画布设计商业模式。 |
| `opc-mvp-designer` | 设计最小可行实验和 MVP 验证路径。 |
| `opc-conversion-loop` | 设计从触达到留资到购买的转化闭环。 |
| `opc-asset-ops` | 将可重复交付结果沉淀为复利资产。 |
| `opc-dashboard-review` | 用轻量指标、瓶颈分析和止损规则复盘经营状态。 |

## Usage

在 Codex 会话中可直接提及 Skill 名称，例如：

- `使用 opc-orchestrator 帮我从零梳理一人企业。`
- `使用 opc-resource-audit 先盘点我的资源。`
- `使用 opc-niche-positioning 分析我的利基市场。`

每个 Skill 的完整说明在对应目录的 `SKILL.md` 中，相关补充材料在 `references/` 中。
