# Directory Guide

## `inbox/`

临时收集箱。适合放未分类的想法、网页摘录、会议片段、待阅读链接和草稿。这里的内容不要求整洁，但应该定期迁移、删除或归档。

## `knowledge/`

长期知识库。这里存放已经整理过、未来会反复引用的知识。它按公司经营主题组织，而不是按文件格式组织。

- `company/`：使命、原则、一人公司模型、长期约束。
- `strategy/`：定位、路线图、护城河、战略选择。
- `products/`：产品想法、规格、定价、反馈。
- `market/`：竞品、趋势、机会。
- `customers/`：用户画像、访谈、问题定义。
- `operations/`：SOP、流程、检查清单。
- `finance/`：收入、成本、指标。
- `legal/`：合同、合规、政策材料。
- `technology/`：技术栈、架构、实验记录。
- `personal/`：学习、复盘、个人生产力。

## `sources/`

外部来源和资料记录。这里回答“这条知识从哪里来”。适合放网页摘录、书籍笔记、论文笔记、播客笔记和未加工原始材料。

## `outputs/`

加工后的产物。适合放研究报告、简报、SOP、Playbook、决策记录、客户可读材料。

## `agent/`

未来知识库 Agent 的准备层。

- `profile/`：Agent 身份、边界、语气。
- `prompts/`：系统提示词和角色提示词。
- `tools/`：工具注册、检索策略、可调用能力说明。
- `memory/`：记忆结构、事实更新规则、长期事实。
- `evals/`：评测问题、期望行为、回归用例。

## `automation/`

辅助知识生产的代码。

- `crawlers/`：网页采集和站点适配。
- `pipelines/`：采集、清洗、转换、导出的流程编排。
- `extractors/`：从 HTML、Markdown、PDF 等内容中抽取结构化信息。
- `normalizers/`：统一 frontmatter、标签、分类和命名。
- `schedulers/`：定时任务或手动任务入口。

## `data/`

自动化流程产生的数据。

- `raw/`：原始抓取结果，尽量不要手改。
- `interim/`：中间结果。
- `processed/`：清洗后的结构化结果。
- `index/`：未来检索索引、嵌入索引或导出索引。

## `templates/`

常用 Markdown 模板。新增知识、来源、决策和评测时优先复制模板。

## `docs/`

仓库自身文档，例如目录规范、命名规范、自动化流程说明和实现计划。

## `config/`

配置文件。可放爬虫目标、标签词表、分类映射、导出参数等。

## `archive/`

过时但仍需保留的内容。迁入前尽量在文件顶部说明归档原因。
