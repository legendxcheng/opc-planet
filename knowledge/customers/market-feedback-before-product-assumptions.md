# 用市场反馈修正产品假设

## Summary

产品无人购买时，优先检查的不是功能是否足够多，而是客户是否真实、痛点是否强、卖点是否清楚、购买场景是否成立。市场反馈比创始人的自我判断更可靠，尤其在一人公司阶段，客户的拒绝、犹豫和成交理由都是产品设计材料。

## Key Points

- 客户痛点不是创始人认为的问题，而是客户愿意为之行动或付费的问题。
- 产品卖点必须从客户语言中提炼，而不是只从专业能力中推导。
- 客户拒绝购买时，要区分是不需要、不信任、不理解、不急，还是价格不匹配。
- 初期不要只看内容数据；收藏、点赞、围观不等于购买意愿。
- 成交后的客户更重要，因为他们提供了真实购买理由和复购线索。

## Details

### 客户反馈四分法

| 反馈类型 | 应该追问 |
| --- | --- |
| 明确购买 | 为什么现在买？替代方案是什么？ |
| 犹豫观望 | 卡在价格、信任、结果、时间，还是交付边界？ |
| 明确拒绝 | 是客户不对，还是痛点不强？ |
| 只互动不购买 | 内容吸引力和交易动机是否错位？ |

### 产品卖点提炼

先不要写“我有什么能力”，而要写：

1. 客户现在在哪个场景里痛？
2. 他原本用什么办法解决？
3. 原办法为什么不够好？
4. 我提供的方案让他少付出什么成本，或多获得什么结果？
5. 他为什么相信我能做到？
