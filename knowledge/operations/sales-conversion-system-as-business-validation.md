# 销转系统是商业验证引擎

## Summary

销转系统不只是销售话术或成交流程，而是验证市场、客户、痛点、价值主张和付费意愿的早期实验室。对一人公司而言，先验证有人愿意为具体承诺付钱，再决定是否做完整产品、内容矩阵或自动化系统。

## Key Points

- 先做产品再找流量，容易把最关键的商业验证推迟到成本最高的时候。
- 销转系统要回答三个问题：市场是否存在、客户是否准确、痛点是否强到愿意付费。
- 最小销转系统可以很简陋：一个价值主张、一条窄渠道、几场真实对话。
- 第一笔钱即使很小，也比点赞、收藏、口头认可更能证明需求。
- 销售访谈不是推销，而是获取客户原话、失败经历、期望方案和付费门槛。

## Details

### 最小销转系统

| 组件 | 作用 |
| --- | --- |
| 价值主张 | 用一句话承诺客户会得到什么结果，而不是描述产品功能。 |
| 窄渠道 | 去潜在客户集中的最小场域，而不是一开始追求全域流量。 |
| 付费对话 | 用一对一诊断、咨询或小额预售验证客户是否愿意付费。 |
| 客户原话 | 记录客户如何描述痛苦、过去踩过的坑和理想解决方案。 |
| 倒推产品 | 用真实付费和真实语言倒推出产品结构、内容和交付。 |

### 为什么先验证销转

产品、内容和流量都会放大前面的判断。如果价值主张错了，产品越完善，沉没成本越高；内容越多，越容易吸引错误用户；流量越大，越容易制造虚假繁荣。销转验证的意义，是在投入过多资源之前把不确定性压低。

## Next Actions

- 为任一新产品先写一句结果承诺，再找 10 个潜在客户做付费或半付费诊断。
- 在做内容矩阵前，先确认客户会不会为这个承诺付钱。
