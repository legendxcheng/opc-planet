# 一人公司表达要站在对方利益点上解释

## Summary

一人公司的文案、汇报和销售，本质上都不是“把事实讲完”，而是站在对方的偏好和利益点上解释：对方想解决什么、在意什么成本、担心什么风险、为什么现在要相信你。只讲事实容易显得正确但无效；能切中对方利益点，才更容易形成理解和行动。

## Key Points

- 做事能力强不等于表达能力强；表达需要考虑对方想听什么、想要什么。
- 文案和销售都要区分“事实正确”与“对方关心”。
- 产品介绍应先回应客户的偏好和利益点，再展开功能细节。
- 一人公司尤其需要这种能力，因为创始人同时承担销售、交付、客服和复盘。
- 结构化表达不是套框架，而是让对方更快判断“这和我有什么关系”。

## Details

### 表达检查表

1. 对方是谁？
2. 对方当前最关心的结果是什么？
3. 对方最担心的成本或风险是什么？
4. 我讲的事实如何对应他的偏好和利益点？
5. 听完之后，对方下一步应该做什么？

### 文案改写方向

| 只讲事实 | 站在对方利益点上解释 |
| --- | --- |
| 这个模板有 10 个模块 | 它把客户、项目、内容和复盘放到同一个系统，减少漏跟进 |
| 我做过很多项目 | 我知道你在这个阶段最容易卡在需求不清和交付边界不清 |
| 这个服务包含 3 次咨询 | 这 3 次分别解决定位、方案和落地复盘，避免你一次性听完却不会做 |
| 我很专业 | 我会先判断你是否适合，不适合会直接告诉你不要买 |
