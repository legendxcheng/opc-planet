# OPC 转化、资产沉淀与经营复盘框架

## Summary

OPC 的运营阶段关注三件事：让客户路径闭环、把重复成果沉淀为资产、用轻量复盘找到下一周期唯一瓶颈。它适合一人公司避免陷入忙碌但不复利的状态。

## Conversion Loop

### Loop Structure

| 环节 | 目标 | 常见形式 |
| --- | --- | --- |
| 触达 | 让目标客户知道你 | 内容、社群、私域、搜索、推荐 |
| 留资 | 让潜在客户表达兴趣 | 表单、私信关键词、预约、问卷 |
| 筛选 | 排除不合适客户 | 资格问题、预算问题、场景问题 |
| 承接 | 建立信任和交付预期 | 诊断、样例、报价、说明页 |
| 成交 | 完成付费和边界确认 | 付款链接、合同、交付说明 |
| 交付后循环 | 形成案例、复购、推荐 | 反馈、复盘、案例授权、转介绍 |

### Channel Strategy

- 主路径只能有一个，避免早期多渠道分散。
- 辅路径用于验证或补充，不应吞噬主要精力。
- 私域适合高信任、高解释成本服务。
- 公域内容适合建立问题认知和筛选潜在客户。
- 表单、问卷和预约页可降低白嫖咨询，提前收集上下文。

## Asset Operations

### Assetization Logic

重复出现三次以上的内容、交付、判断、流程或问题，都应该考虑资产化。

### Asset Types

- 内容资产：高频问题解答、案例、教程、观点文章。
- 交付资产：报告模板、检查清单、SOP、报价结构。
- 产品资产：小工具、自动化脚本、表单系统、轻量应用。
- 数据资产：客户问题库、案例库、指标记录。
- 关系资产：私域名单、合作渠道、推荐网络。
- 知识资产：方法论、决策记录、复盘结论。

### Priority Criteria

优先沉淀：

- 高频出现。
- 复用后能明显省时间。
- 能提高成交或交付质量。
- 能降低对个人临场状态的依赖。
- 能成为未来产品或 Agent 的基础。

## Dashboard Review

### Lightweight Metrics

- 触达：内容发布、曝光、私域触点、推荐次数。
- 转化：留资数、有效咨询数、成交率。
- 收入：订单数、客单价、收入、复购。
- 交付：单客耗时、交付周期、返工率。
- 资产：新增模板、案例、自动化、SOP。

### Bottleneck Types

- 触达瓶颈：客户不知道你。
- 信任瓶颈：客户看到但不相信。
- 价值瓶颈：客户不理解为什么要买。
- 价格瓶颈：客户觉得不值或不愿现在买。
- 交付瓶颈：成交后过度消耗个人时间。
- 战略瓶颈：选错人群、问题或渠道。

### Review Output

每次复盘只确定一个下一周期优先重点。不要同时优化所有指标。
