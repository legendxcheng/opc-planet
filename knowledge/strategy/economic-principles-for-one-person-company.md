# 一人公司要把经济学概念转成经营约束

## Summary

一人公司不能只靠热情、勤奋或单点技能推进，而要把机会成本、边际增量、比较优势、MVP、定价锚点、客户筛选和边际成本等经济学概念转成日常经营约束。经济学在这里不是理论装饰，而是一组防止裸辞冲动、错误定价、无效扩张和低质量交付的检查问题。

## Key Points

- 机会成本要求创始人明确：做这个方向，就意味着放弃哪些更高回报或更低风险的选择。
- MVP 的重点不是做一个粗糙产品，而是用最低成本验证最关键的交易假设。
- 比较优势比“绝对擅长”更适合一人公司：选择自己相对更低成本、更快反馈、更容易积累资产的切口。
- 定价不是最后一步，而是筛选客户、管理预期、定义交付边界和验证价值强度的核心机制。
- 一人公司不适合正面硬刚大公司，应寻找配套服务、信息筛选、时间节省、桥梁型服务等市场缝隙。
- 长期壁垒来自低边际成本产品与不可复制服务的组合，而不是持续堆创始人的手工时间。

## Details

### 从经济学概念到行动问题

| 概念 | 一人公司里的行动问题 |
| --- | --- |
| 边际增量 | 下一小时、下一条内容、下一次销售动作，能增加什么可观察结果？ |
| 隐藏信息 | 建议者、竞品、客户和平台有哪些背景条件没有被说出来？ |
| 定义胜利 | 这轮实验到底以成交、留资、复购、案例还是学习为胜利？ |
| MVP | 最小实验能否验证客户愿不愿意付钱，而不只是愿不愿意点赞？ |
| 机会成本 | 当前方向占用的时间，是否挤掉了更适合自己的机会？ |
| 比较优势 | 我在哪类客户、交付、表达或渠道上有相对成本优势？ |
| 锚定效应 | 报价前是否让客户理解结果价值、替代成本和节省的时间？ |
| 诱饵选项 | 套餐结构是否帮助客户做选择，而不是制造混乱？ |
| 价格歧视 | 是否能用不同版本服务不同支付能力和需求强度的客户？ |
| 预期管理 | 成交前是否清楚说明结果边界、交付方式和不承诺的部分？ |
| 外部性 | 当前获客动作是在积累信任，还是在消耗品牌口碑？ |
| 客户筛选 | 价格、内容和销售话术是否主动排除不适合的客户？ |

### 对 OPC 阶段的补充

| OPC 阶段 | 可补充的经济学约束 |
| --- | --- |
| 资源盘点 | 识别比较优势和机会成本，不把“我会什么”直接等同于“我该卖什么”。 |
| 利基定位 | 避开正面战场，寻找配套服务、信息筛选、时间节省和桥梁型机会。 |
| 价值主张 | 明确客户节省什么成本、减少什么风险、获得什么确定性。 |
| 商业模式 | 用价格梯度、预期管理和客户筛选保护利润与交付边界。 |
| MVP | 用最小实验验证需求强度、付费意愿和渠道反馈。 |
| 转化闭环 | 让免费内容负责信任和解释，让付费服务负责诊断、陪伴和结果。 |
| 资产沉淀 | 把高频问题沉淀为问题库、SOP、课程、模板和社群交付机制。 |
| 经营复盘 | 用边际利润、时间占用、客户质量和外部性判断是否继续扩张。 |

### 使用边界

这条来源有强烈营销化表达，适合提炼经营检查问题，不适合逐字当作原则。尤其是税务、开公司、全球套利和具体利润阈值相关内容，必须另查权威法规后才能进入操作建议。
