# OPC 利基定位与价值主张框架

## Summary

OPC 的市场选择不是先问“我想做什么”，而是寻找新杠杆、边界变动和个人资源的交集。确定利基后，再用 Jobs / Pains / Gains 将目标客群的真实需求转成价值主张。

## Niche Positioning

### Three-Ring Analysis

| 环 | 关注点 | 判断问题 |
| --- | --- | --- |
| 新杠杆 | 新工具、新平台、新分发、新自动化能力 | 什么新能力让过去做不了或成本太高的事变得可行？ |
| 边界变动 | 规则、平台、成本、需求、供给、客户行为变化 | 哪些变化让旧方案失效或新需求出现？ |
| 独有资源 | 经验、人群、能力、关系、渠道、资产、约束 | 我凭什么比别人更容易切入这个点？ |

### Opportunity Evaluation

候选利基至少从这些维度评分：

- 需求强度：痛点是否真实、频繁、紧急。
- 付费意愿：客户是否愿意为解决方案付钱。
- 可触达性：能否低成本接触到目标客户。
- 资源匹配：是否匹配已有经验、能力、渠道和资产。
- 差异化：是否能形成清晰定位，而不是泛泛竞争。
- 验证成本：能否用小实验快速验证。

### Positioning Statement

推荐格式：

> 我帮助 [目标人群] 在 [关键场景] 中解决 [高强度问题]，通过 [独特方法/资源/杠杆] 获得 [明确结果]，区别于 [替代方案] 的地方是 [差异化]。

## Value Proposition

### Customer Profile

- Jobs：客户想完成的功能任务、情感任务和社会任务。
- Pains：客户的损失、风险、麻烦、焦虑和失败成本。
- Gains：客户想得到的收益、确定性、效率、体验和身份感。

### Value Map

- Products / Services：你实际提供什么。
- Pain Relievers：如何降低风险、麻烦或损失。
- Gain Creators：如何创造收益、确定性或更好体验。

### Messaging Options

价值主张可生成多个版本：

- 降风险型：强调避坑、止损、确定性。
- 结果型：强调可见结果和达成路径。
- 效率型：强调省时间、省精力、省决策成本。
- 双核心型：同时呈现风险降低和结果达成。

## Common Mistakes

- 只描述自己会什么，不描述客户为什么现在需要。
- 把“大市场”误认为“好利基”。
- 没有明确替代方案，导致差异化无法成立。
- 过早进入产品功能，而不是先确认客户任务和付费动机。
- 价值主张过泛，无法指导内容、报价和 MVP。
