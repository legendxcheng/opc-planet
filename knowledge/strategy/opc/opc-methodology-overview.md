# OPC 方法论总览

## Summary

OPC 是一套把个人资源转化为一人企业的阶段化方法论。它强调先盘点资源，再选择利基，随后设计价值主张、商业模式、MVP 和转化闭环，最后通过资产沉淀和经营复盘形成复利。

## Core Idea

OPC 的核心不是“一个人做所有事”，而是用清晰阶段、文件化产出和边界控制，把个人能力组织成可验证、可复用、可迭代的业务系统。

## Workflow

| 阶段 | 名称 | 关键问题 | 阶段性质 |
| --- | --- | --- | --- |
| 00 | 总编排 | 当前处在哪一步？该调用哪个模块？ | 编排层 |
| 01 | 资源盘点 | 我手上到底有什么？哪些不能碰？ | 线性阶段 |
| 02 | 利基定位 | 哪个细分机会最值得切入？ | 线性阶段 |
| 03 | 价值主张 | 为谁解决什么问题，凭什么买？ | 线性阶段 |
| 04 | 商业模式设计 | 如何收费、获客、交付和衡量？ | 线性阶段 |
| 06 | MVP 设计 | 用什么最小实验验证最大风险？ | 线性阶段 |
| 07 | 转化闭环 | 从触达到成交如何跑通？ | 线性阶段 |
| 08 | 资产沉淀 | 哪些成果能复用并产生复利？ | 运营触发 |
| 09 | 经营复盘 | 当前瓶颈是什么，下一周期只抓什么？ | 运营触发 |

## Principles

- 不跳步：下游判断必须依赖上游产物。
- 一次只问一个关键问题：降低用户认知负担。
- 先广后深：先建立全局，再钻取细节。
- 用户确认后落盘：重要产物必须写成文件。
- 阶段边界清晰：不要在资源盘点时提前做方向判断，不要在 MVP 阶段提前做规模化运营。
- 资产优先：运营过程中持续寻找可复用、可模板化、可自动化的结果。

## Stage Boundaries

- 总编排不替用户完成所有分析，而是判断阶段、恢复进度、调用对应 Skill。
- 资源盘点只确认事实资源，不分析偏好、方向、风险承受度。
- 利基定位选择细分市场，不设计商业模式和转化路径。
- 价值主张聚焦客户任务、痛点、收益和表达，不提前定价。
- 商业模式确定收费结构、交付结构和风险假设，不设计 MVP 细节。
- MVP 只验证关键假设，不追求完整产品。
- 转化闭环关注触达、留资、承接、成交，不做大规模增长。
- 资产沉淀和经营复盘只在运营期或卡点触发。
